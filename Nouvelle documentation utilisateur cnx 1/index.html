Introduction

CNX est une extension de Business Central qui vous permet d'avoir une meilleur contrôle sur le flow d'importation et d'exportation de vos données. Si l'un de vos système offre un API ou que vous êtes capable d'exporter des données sous format XML ou JSON alors CNX peux vous aider à bâtir un pipeline entre vos autres système et Business Central.

Le processus d'automatisation est le suivant.

  1. Créer un data flows
  2. Créer un définition d'échange de données
  3. Créer un configuration package
  4. Créer une file d'attente des travaux

Data flows

Le cœur de CNX sont les Data flows. C'est des fiches qui vous permette de lier la bonne définition d'échange de données avec le bon configuration package et d'affecter la bonne file d'attente des travaux.

Ouvrez la page Data Flows pour avoir la liste de vos flux de donnée.

Nouveau

  1. Ouvrir la page Data Flows
  2. Utiliser la fonction Nouveau >> Nouveau
  3. Donnée un code et un nom au flux de données
  4. Cocher les cases d'autoprocess si vous souhaiter traiter les données immédiatement après réception sinon voir processus Traiter un data flow
  5. Sélectionner le bon Data Exchange définition. Si aucun n'est adéquat voir le processus Ajouter une définition d'échange de données
  6. Attribuer une Job Queue Category
  7. Utiliser la fonction Nouveau >> Create Config. Package
  8. Ajouter les informations sous la section API Configuration si nécessaire

Traiter un data flow

Parfois, il est possible de traiter manuellement un data flow. C'est utile dans le processus de développement et de configuration. Ainsi, vous pouvez modifier la source de donnée ou la configuration de l'échange de donnée afin de traiter les données incorrectement traiter.

  1. Ouvrir le data flow qui vous intéresse
  2. Utiliser la fonction Nouveau >> Process Instance
  3. Appuyer sur OK

Si une erreur survient voici la procédure pour corriger le problème. 1. Ouvrir le data flow fautif 2. Vérifier les entrer de journal avec la fonction Associé >> Log entries 3. Utiliser la fonction Nouveau >> Download file pour obtenir le dernier jeux de donnée traiter 4. Ouvrir le fichier télécharger 5. Corriger la donnée ou identifier le format de la donnée qui cause l'erreur 6. Utiliser la fonction Actions >> Import file 7. Utiliser la fonction Nouveau >> Process Instance 8. Data Exchange définition

Ajouter une définition d'échange de données

  1. Ouvrir la page Définitions d'échange de données
  2. Utiliser la fonction Nouveau
  3. Sous la section Général
    1. Donnée un code et un nom à la définition d'échange de données
    2. Choisir un type de fichier et utiliser le type Importation générique
    3. writing codeunite Voir table
  4. Sous la section Définition ligne
    1. Entrer le détail de la ligne, le code et le Nom
    2. Fournir l'étiquette de ligne de données
    3. Utiliser la fonction Gérer >> Correspondance de champ
    4. Voir le processus Correspondance champ une fois les définitions colonne terminer
  5. Sous la section Définitions Colonne
    1. Pour chaque Définitions de lignes, entrer le no de colonne, le nom et le type de données
    2. Entrer le Chemin et autre information

Correspondance champ

  1. Sur la page correspondance champ
  2. Sous la section Général
    1. Entrer le code de table ainsi que le nom
    2. Sélectionner le codeunit si besoin
  3. Dans la section Correspondance de champ
    1. Inscrire le no de colonne de votre source de donnée
    2. Inscrire le code champ qui recevra l'information dans business central
    3. Si nécessaire définisse une règle de transformation
    4. Si pertinent, ajouter le code de dimension associer

Ajouter une règle de transformation

  1. Ouvrir la page Règles de transformation
  2. Utiliser la fonction Nouveau
  3. Donner un nom et une description
  4. Utiliser le type de transformation ou une règle de transformation
  5. Pour tester la règle, entrer une donnée test dans la boite Texte du test, puis cliquer Mettre à jour Vous pouvez utiliser les expressions régulières afin d'opérer les bonnes transformation sur les texte de donnée : Expression régulière — Wikipédia (wikipedia.org)

Ajouter un package de configuration

  1. Sur la carte de data flow, utiliser la fonction Create configueration package

Priorité champ

  1. Ouvrir un configuration package
  3. Sous la section Tables
    1. Sélectionner une lignes
    2. Utiliser la fonction Table >> Champs
    3. Cliquer sur modifier la liste

Flux de travail

L'une des fonctionnalités de Business Central c'est les flux de travail Automatisation lors des modifications exemple

  1. Ouvrir la page Flux de travail
  2. Utiliser la fonction Nouveau >> Créer
  3. Entrer un code, une description et une catégorie
  4. Sous la section Étapes de flux de travail
    1. Sélectionner le cas d'événement A specific field on a record has received a modification
    2. Cliquer sur la condition
    3. Ajouter les filtres TableNo et No et entrer les numéros associer
    4. Cliquer sur la réponse
    5. Sélectionner la réponse Schedule and API request through the job queue to sent at a determined time ou Send an API request to a configured external system
    6. Sélectionner le data flow CNX, puis sur OK
  5. Activé le flux de travail
